Full Service Flooring - How To Run Code

  1. Locate the project folder and move it into the htdocs directory within your XAMPP installation.
  2. Open a web browser and navigate to http://localhost/dashboard, then access phpMyAdmin.
  3. Create a new database named fullserviceflooring.
  4. Open the database.sql file located in the project folder, copy its contents, and paste them into the SQL tab in phpMyAdmin to generate the required tables.
  5. Navigate to the project folder and locate the uploads folder.
  6. Update the sharing and permissions of the uploads folder to allow read and write access for all users.
  7. On macOS, right click the uploads folder in Finder, select “Get Info,” and adjust the Sharing and Permissions settings to Read and Write for all listed users. Administrator access may be required.
  8. On Windows, modify the folder permissions to allow read and write access. Refer to your system’s file permission settings if needed.
  9. Open the XAMPP Control Panel and start both the Apache Web Server and MySQL Database services.
  10. Open a web browser and enter the following URL: http://localhost/ProjectOne/index.html
  11. The Full Service Flooring website should now load successfully.

Preptastic - Meal Planning Website

Preptastic is a full stack meal planning web application designed to help users organize and manage their meals efficiently. The platform allows users to create and post recipes, browse recipes from others, save favorites, schedule meals by date and time, and generate ingredient lists.


This project was built using HTML, CSS, JavaScript, and PHP. My main contributions focused on developing the recipe creation functionality, implementing the saved recipes feature, and building the recipe browsing page. I also contributed to the design and user experience of these sections to ensure they were both intuitive and visually appealing.

Preptastic - How To Run Code

  1. Install a local server environment such as XAMPP if it is not already installed on your system.
  2. Open the XAMPP control panel and start the Apache, MySQL, and ProFTPD services.
  3. Locate the project folder named Preptastic-1 and place it inside the htdocs directory within your XAMPP installation.
  4. Open a web browser and navigate to http://localhost/phpmyadmin.
  5. Create a new database named preptastic.
  6. Set up the database connection in your PHP files using the following configuration:

    $db_server = "localhost";
    $db_user = "root";
    $db_pass = "";
    $db_name = "preptastic";

  7. Open the SQL tab in phpMyAdmin and copy and paste the provided table creation scripts.
  8. Run the scripts in the following order to ensure proper setup:

    First create the recipes table and then apply the ALTER TABLE statement to add the posted_to_feed column.
    Next create the ingredient table.
    Finally create the recipe_calendar table with the foreign key relationship.

  9. If you would like sample data to appear on the feed page, copy and paste the provided recipe INSERT statements after the tables have been created.
  10. Confirm that all tables and data have been successfully added to the preptastic database.
  11. Once the database setup is complete, return to the XAMPP control panel and verify that all required services are still running.
  12. Open a web browser and enter the following URL:

    http://localhost/Preptastic-1/Calendar1.html
  13. If your project files are located within additional subfolders, include those folder names in the URL path as needed.
  14. The Preptastic website should now load and be fully functional.

Faculty and Student Information System - Computer Security Project

This project was developed as part of a computer security course and focuses on role based access control within a web application. Built primarily using Node.js, the system simulates a dashboard that displays different functionality depending on the user’s access level.


There are two main roles in the system: student and faculty. Students can view and update their personal information, such as their address, while faculty members have extended permissions. Faculty users can modify student records including tuition and GPA, as well as create new student and faculty accounts.

Security Project - How To Run Code

  1. Install the required software before starting the project. This includes Node.js for running the backend server and XAMPP for hosting the MySQL database.
  2. Open the XAMPP Control Panel and start both the Apache and MySQL services.
  3. Open Visual Studio Code, select “File,” then “Open Folder,” and choose the folder containing the project files.
  4. If any files appear with names such as old_fileName.fileType, they can be ignored as they are not required for execution.
  5. In the XAMPP Control Panel, click “Admin” next to MySQL to open phpMyAdmin.
  6. Locate the sql.txt file in the project folder, copy its contents, and paste them into the SQL tab in phpMyAdmin to create the required database and tables.
  7. Return to Visual Studio Code and open the integrated terminal.
  8. Install the required Node.js dependencies by running the following commands:

    npm install
    npm install bcrypt
    npm install cookie-parser
    npm install crypto
    npm install axios
    npm install body-parser
    npm install nodemailer

  9. Start the backend server by running the following command in the terminal: node server.js
  10. After the server starts, a message will appear in the terminal indicating the server is running at a local URL such as https://localhost:3000/.
  11. Copy the provided URL and open it in a web browser.
  12. Append /login.html to the end of the URL so that the login page loads first. For example: https://localhost:3000/login.html
  13. To log in as a faculty user, use the following credentials:
    Email: faculty1@uwec.edu
    Password: faculty
  14. After logging in as a faculty member, the Faculty Dashboard will appear. From this page, users can view personal information, create new student or faculty accounts, and manage student academic data. Use the logout button when finished.
  15. To log in as a student, use the following credentials:
    Email: fakestudent1@uwec.edu
    Password: student
  16. After logging in as a student, the Student Dashboard will appear. From this page, users can update personal information and view academic records. Students do not have permission to edit academic data.
  17. If desired, follow the optional steps below to configure a trusted HTTPS certificate and remove browser security warnings.
  18. Navigate to the cert folder in the project and delete the existing certificate files named localhost.pem and localhost-key.pem.
  19. On macOS systems, install Homebrew if it is not already installed.
  20. In the terminal, install mkcert and its dependencies by running:
    brew install mkcert
    brew install nss
  21. Generate new local certificates by running: mkcert -install
  22. Move the newly generated localhost.pem and localhost-key.pem files into the project’s certs folder.
  23. Restart the server using node server.js and navigate to: https://localhost:3000/login.html
  24. If the browser still marks the site as not secure, try accessing it in a different browser or in incognito mode.
  25. To resolve this issue in Google Chrome, stop the server, open Chrome, and navigate to: chrome://net-internals/#hsts
  26. Under “Delete domain security policies,” enter localhost and delete the entry.
  27. Fully close and restart Chrome. The site should now appear as secure.
